## 🧠 核心概念解析

### 1. 什麼是「自主計算智能體 (Computational Agent)」？
*   **定義**：感知環境、自主（Autonomously）採取行動以實現目標，並在運行過程中不斷優化性能的計算實體。
*   **發展脈絡**：智能體概念並非近期產物，自 1940 年代起即有相關理論研究，並深受複雜系統（如生命遊戲 Game of Life、簡單局部規則交互演化出複雜行為）的啟發。
*   **基本運行循環**：
    ```
    感知環境 (Sense) ➡️ 分析數據 (Analyze) ➡️ 做出決策 (Decide) ➡️ 執行行動 (Act)
    ```

### 2. 智能體的解剖結構 (Anatomy of an Agent)
講者強調：**智能體遠不止於語言模型（LLM）本身**。一個完整的智能體系統由以下要素組成：
*   **技能 (Skills)**：指導智能體如何執行特定任務的規則與說明。例如「如何查詢結構化數據」，包含目的摘要、用例與輸入輸出規範。
*   **工具 (Tools)**：智能體可調用的 API 或腳本（例如 scikit-learn、Python 運行環境）。
    *   **MCP (Model Context Protocol)**：微軟等推廣的開放標準，定義了智能體可用的工具列表，使智能體能直接使用資料庫或 GitHub 資源，免去讓 LLM 重複寫代碼的 token 浪費。
*   **知識 (Knowledge)**：外部知識庫的整合。例如講者團隊開發的 **阿爾茨海默病知識庫 (Alzheimer knowledge base)**，整合了 20 多個知識來源與生物醫學本體論，避免智能體受限於模型預訓練的知識盲區。
*   **推理 (Reasoning)**：由 LLM 或其他 AI 模型提供的推導能力。
*   **角色 (Persona)**：定義智能體的決策傾向與風格（如風險偏好者、保守派）。
*   **行動 (Action)**：與其他智能體通信、傳遞程式碼、發送簡訊、更新資料庫或在 GitHub 上發布分析結果。

## 🛠️ 智能體工程 (Agent Engineering) 與制衡機制

### 1. 線束與環境 (Harness & Environment)
*   **線束 (Harness)** 是指智能體運作的基礎設施與環境。它負責將一個複雜的科學問題拆解為數個子任務，並為智能體提供執行與測試的虛擬環境。
*   近年來研究已從「提示詞工程 (Prompt Engineering)」轉向「智能體工程 (Agent Engineering)」，強調透過多智能體環路交互疊代，讓系統自動演化出最佳解決方案。

### 2. 制衡與監督機制 (Checks and Balances)
為了解決 LLM 容易產生幻覺或犯錯的問題，在設計智能體系統時，必須加入制衡機制：
*   **經理智能體 (Manager Agent)**：負責任務分配與全局調度。
*   **品質控制智能體 (Quality Agent)**：驗證所提取或分析的數據是否正確且符合標準。
*   **倫理智能體 (Ethics Agent)**：審查智能體提出的行動方案是否符合倫理規範。
*   **模式檢查器 (Pattern Checker Script/Agent)**：使用輕量級腳本驗證智能體的工具調用路徑與輸出結果是否正確，建立可信賴的運作鏈。

## 📊 案例研究與實務應用

### 💼 案例一：出院患者隨訪 (Patient Discharge Follow-up)
*   **傳統做法**：單一 LLM 接收提示詞後直接給出建議（易犯錯、限制多）。
*   **智能體團隊架構**：
    1.  **專科智能體 A (數據準備)**：運行 SQL 查詢電子健康記錄 (EHR)，結合臨床本體論準備數據。
    2.  **專科智能體 B (數據分析)**：具備統計與機器學習知識，使用 Python/scikit-learn 進行建模分析。
    3.  **專科智能體 C (決策解讀)**：將分析結果轉化為臨床建議（例如決定是否給病人打電話隨訪，並擬定通話腳本）。
    4.  **管理層**：經理、品質與倫理智能體共同審查，確保方案安全可信。

### 🧪 案例二：智能體自主進行科學研究 (AI Scientist)
*   **運作流程**：
    1.  經理智能體利用夜間時間自動閱讀文獻並歸納結論。
    2.  提出研究假設，並與本地數據集比對以規劃分析方案。
    3.  進行公平性與偏見分析（Fairness and Bias Analysis）。
    4.  產出數據分析結果、圖表，並在翌日清晨自動撰寫好論文草稿。
*   **講者實測**：使用訂閱制商業科研智能體工具，輸入臨床問題後，智能體在 **90 分鐘內** 完成了「患者數據比對（10k 樣本） ➡️ 生成兩頁包含迴歸方程的分析計劃 ➡️ 執行分析產出圖表 ➡️ 撰寫 JAMA 格式論文」的全套流程。
*   **商業工具的缺點**：運作過程為「黑盒子」，不公開智能體的決策軌跡與原始碼。科學界需要開源、可重現的解決方案。
*   **團隊開源成果**：講者開發了一款自主 AI 科學研究系統並提交至 bioRxiv。該系統包含可視化界面，展示智能體在虛擬空間中的移動、在「茶水間」交流資訊、在數據中心存取資料的動態。

### 🧫 案例三：實驗室自動化 (Proteomics Lab Automation)
*   講者同事（邁爾斯）成功自動化了其蛋白質組學實驗室。
*   實驗室配備高通量機器人（96/384 孔板）。質譜儀產出 raw data 後，智能體團隊會自動接管數據處理、統計分析與論文撰寫。
*   **成效**：以前 PhD 學生需要花費數月甚至數年完成的研究，現在系統可以在**數天內**完成 100 至 200 種藥物的完整質譜分析與論文草稿。

### 📹 案例四：PDF 論文自動生成短影音 (n8n 平台)
*   **n8n 平台**（原始逐字稿誤植為 "N"）：講者開發基於 n8n 的智能體工作流，用戶只需上傳 PDF 論文連結 ➡️ 智能體團隊自動理解論文、撰寫影片劇本 ➡️ 調用影片生成工具製作出帶有虛擬頭像 (Avatar) 的總結短影音（製作時間小於一小時）。
