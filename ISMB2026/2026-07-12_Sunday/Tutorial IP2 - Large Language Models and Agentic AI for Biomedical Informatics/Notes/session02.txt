5. 建立權重共享 Supernet
這一段是整份教學的核心。

pool = nn.ModuleDict()
建立一個模組池，存放所有不同的神經網路區塊。

建立唯一模組名稱
def _k(t, i, o):

例如：

cnn_64_128
同樣類型、同樣輸入輸出維度的模組，會使用相同名稱。

避免重複建立模組
for p in PATHS:
    for c in p:
        if _k(*c) not in pool:
            pool[_k(*c)] = build_module(*c)

("transformer", 128, 128)
程式不會建立三個 Transformer，而只建立一個：

這就是 權重共享 Weight Sharing。

建立每條模型路徑
paths = [
    nn.Sequential(*[pool[_k(*c)] for c in p])
    for p in PATHS

將每條候選架構組成 nn.Sequential。

例如第一條路徑會類似：

6. 建立 Masked Language Model 資料
def mb(bs=8, Lb=32):
這個函式每次產生一個訓練 batch。

參數：

bs=8：一次 8 條 DNA

Lb=32：每條最多取 32 個 token

隨機抽取 DNA
ids = [
    torch.tensor(tok.encode(s).ids[:Lb])
    for s in random.sample(seqs, bs)
]
從 96 條 DNA 中隨機抽出 8 條，並且只取前 32 個 token。

補齊序列
ids = torch.nn.utils.rnn.pad_sequence(
    ids,
    batch_first=True,
    padding_value=PAD
)
將不同長度的序列補齊成相同長度。

輸出形狀大致為：

8 × 32
建立答案與遮罩位置
tgt = torch.full_like(ids, PAD)
m = torch.rand(ids.shape) < 0.15
m 是一個布林矩陣，每個位置有 15% 機率為 True。

tgt[m] = ids[m]
只有被遮住的位置會保留正確答案。

其他位置保持 [PAD]，之後 loss 會忽略它們。

ids[m] = MASK
將選中的 15% token 替換成 [MASK]。

模型前向傳播
def run(pi, ids):
    return head_s(paths[pi](emb_s(ids)))
執行流程：

token IDs
  ↓
Embedding
  ↓
選定的 path
  ↓
Linear prediction head
pi 是路徑編號，例如：

run(2, ids)
表示使用第 2 條候選架構。

Loss 函式
lossfn = nn.CrossEntropyLoss(ignore_index=PAD)
使用交叉熵計算 token 預測錯誤。

ignore_index=PAD 表示答案為 [PAD] 的位置不計算 loss。

所以只有被遮罩的 15% 位置會影響訓練。

7. One-shot 預訓練
opt = torch.optim.AdamW(
    list(emb_s.parameters())
    + list(pool.parameters())
    + list(head_s.parameters()),
    lr=1e-3
)
AdamW 會更新：

Embedding

共享模組池

預測 head

每一步只訓練一條路徑
for step in range(60):
    ids, tgt = mb()
    pi = random.randrange(len(paths))
每一步：

建立一個 DNA batch

隨機選擇一條架構

loss = lossfn(
    run(pi, ids).permute(0, 2, 1),
    tgt
)
假設模型輸出原本是：

batch × sequence × vocab
但 CrossEntropyLoss 需要：

batch × classes × sequence
因此使用：

.permute(0, 2, 1)
將維度改成：

batch × vocab × sequence
反向傳播
opt.zero_grad()
loss.backward()
opt.step()
標準 PyTorch 訓練流程：

清除舊梯度
→ 計算新梯度
→ 更新模型參數
雖然每一步只選一條路徑，但多條路徑共享部分模組，所以某條路徑更新後，其他使用相同模組的路徑也可能一起改善。

這就是 one-shot NAS 可以降低搜尋成本的原因。

程式中值得注意的問題
1. 註解與實際測試集數量不一致
程式寫：

N_TE = 1000
但後面註解是：

# keep the SAME 500-row test subset
實際上保留的是最多 1,000 筆，不是 500 筆。這應該是舊註解沒有更新。

2. PICK = CASE_IDS[4] 不安全
若案例不足 5 筆會報錯，建議改成：

if len(CASE_IDS) == 0:
    raise RuntimeError("沒有可供比較的案例")

PICK = CASE_IDS[min(4, len(CASE_IDS) - 1)]
3. NT-50M 的 learning rate 可能偏大
LR_NT = 5e-4
對完整微調 50M pretrained Transformer 而言，5e-4 可能偏高。

可考慮：

LR_NT = 1e-5
或：

LR_NT = 5e-5
但最佳值仍需實驗確認。

目前設定可能使 NT-50M 在小資料集上較容易破壞預訓練權重，因此比較結果不一定完全公平。

4. 使用測試集挑最佳 Epoch
程式中的：

best = max(best, acc())
每個 epoch 都在 test set 上計算，最後選 test accuracy 最高的一次。

嚴格研究流程應該分成：

Train：更新模型
Validation：挑選最佳模型與超參數
Test：最後只評估一次
否則會間接對 test set 過度調整。

5. 沒有保存最佳 Epoch 的模型權重
程式只保存最高 accuracy：

best = max(best, acc())
但 m 或 clf 最後回傳的是最後一個 epoch 的模型，不一定是 accuracy 最佳時的模型。

可以改成：

best_acc = 0
best_state = None

current_acc = acc()

if current_acc > best_acc:
    best_acc = current_acc
    best_state = copy.deepcopy(m.state_dict())

m.load_state_dict(best_state)
這樣後面的 interpretation 才確定使用最佳模型。
